## 資料庫的兩大分類

資料庫可以分成以下兩大類：

1. 關聯式資料庫 (Relational Database / SQL Database)
2. 非關聯式資料庫 (Non-relational Database / NoSQL Database)

### 關聯式資料庫 (Relational Database / SQL Database)

- 數據以表格的形式呈現，表格即是欄（直 / column）和列（橫 / row）組成，前者稱為「屬性 (Attribute)」，後者稱「記錄 (Record)」。
- 各表格之間可以有關係。
- 關聯式資料庫中的查詢和操作都是使用 "SQL (SeQuel)"(Structure Query Language)。
- 亦稱為「通用資料庫 (General Purpose Databases)」，是目前最廣泛用來建立系統和應用程式的資料庫。
- 市面上主流的關聯式資料庫有：
  - MySQL
  - PostgreSQL
  - Oracle
  - Microsoft SQL server
  - 基於雲端的關聯式資料庫有：
    - Amazon Aurora
    - Google Cloud Spanner
    - Azure SQL DB

由於發展越來越快，應用程式也越趨複雜，建立以及維護大型且複雜的資料庫是非常耗費資源和成本，因此市場上也出現了替代的資料庫類型，也就是「非關聯式資料庫」。

### 非關聯式資料庫 (Non-relational Database / NoSQL Database)

NoSQL 資料庫包括：
- 鍵值資料庫 (Key-Value DB)
- 記憶體資料庫 (In-memory DB)
- 圖學資料庫 (Graph DB)
- 文件資料庫 (Document DB)

#### 鍵值資料庫 (Key-Value DB)

- 採用鍵值組集合，可水平擴充。
- 此類型的資料庫最多。
- 每筆資料都是獨立的，因此具備高度分散跟擴充性。
- 常見的有：Google Big Table, Amazon Dynamo 等。
